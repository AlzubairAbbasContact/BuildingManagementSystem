# نظام إدارة المباني (BMS)

تطبيق PHP بسيط بنمط MVC لإدارة الشقق، المستأجرين، والمدفوعات.

نظرة عامة
- اللغة: PHP 8.x
- البنية: MVC بسيط (مجلدات `app/Controllers`, `app/Models`, `views/`)
- قاعدة البيانات: MySQL / MariaDB (انظر `bms_db.sql`)

الميزات
- إدارة الشقق (حالة: شاغرة/مؤجرة)
- إدارة المستأجرين (إضافة، إنهاء العقد)
- إدارة المدفوعات (تسجيل دفعات، إلغاء مع سبب)
- سلوك حذف رخوة: السجلات محفوظة عبر حقول الحالة (`status`)
- الحسابات المالية ديناميكية: لا نعتمد على حقل "المتبقي" المخزن

بدء سريع
1. انسخ المشروع إلى مجلد الويب (مثل `htdocs` في XAMPP).
2. أنشئ قاعدة بيانات MySQL (مثلاً `bms_db`) واستورد `bms_db.sql`.
3. نفّذ ملف الترحيل `migrations/20260223_add_status_and_constraints.sql` لإضافة أعمدة `status` وتعديل المفاتيح المرجعية.
4. عدّل إعدادات الاتصال في `app/Config/config.php` لتتناسب مع بياناتك.
5. اجعل جذر المستندات في الخادم هو مجلد `public/` وافتح التطبيق عبر المتصفح.

هيكل المشروع
- `app/Models` — نماذج قاعدة البيانات (Apartment, Tenant, Payment, User)
- `app/Controllers` — منطق التحكم وعمليات الـ HTTP
- `views/` — واجهات العرض (قوالب PHP)
- `public/` — مدخل التطبيق وموارد عامة
- `migrations/` — ملفات SQL للترحيل

سلوك مهم
- المتبقي يُحسب دائماً: `remaining = tenant.rent_amount - SUM(active payments)`
- إلغاء دفعة: `payments.status = 'canceled'` ويمكن تخزين `cancellation_reason`
- إنهاء عقد مستأجر: `tenants.status = 'inactive'` وتعيين الشقة `vacant`
- حذف الشقة يجعلها شاغرة فقط (لا حذف مادي)
